# Aletheos Minimal Apparatus 001

Status: pre-ratification technical profile.

## Thesis

Boundary before body.

Witness before authority.

Receipt before trust.

Aletheos does not begin as a dictionary of meanings or a certificate registry.

Aletheos begins with the condition under which any meaning, certificate, claim, or symbol can become accountable.

## Root condition

Within the Aletheos binding lane:

    0 = NULL_WELL
    1 = ALETHEOS_ROOT

Index 0 is the unbound well.

Index 1 is the witness root.

The Aletheos Root defines the minimal apparatus through which a symbol may cross from unbound state into receipted witness.

## Minimal apparatus

An Aletheos-bound apparatus is a bounded state-transition system that can:

1. distinguish inside from outside;
2. hold state;
3. apply logic;
4. interpret instruction language;
5. sequence admissible operations;
6. preserve memory;
7. cross an input/output boundary;
8. produce a receipt of admitted return;
9. allow a symbol to acquire body only after witness.

## Apparatus order

The conceptual order is:

    null well
    boundary
    state
    logic
    language
    sequence
    memory
    input/output
    receipt
    body

This order is not a B32K index assignment.

It is an apparatus grammar under ALETHEOS_ROOT.

## Operator connection

The minimal apparatus is compatible with the operator profile:

    A = local motion
    M = admission
    M^T = return
    Q = receipt

The primary receipt law remains:

    Q = M M^T

An Aletheos-bound object is not trusted merely because it is present.

It must pass through admission and return as a reproducible receipt.

## Keeper

Boundary before body.

Witness before authority.

Receipt before trust.

Aletheos begins by drawing a boundary around witness.

Only then may a symbol acquire body.
